JIMMY — AI-Powered Microservices Platform
Autonomous Incident Response · Python · AWS Bedrock · EKS · PostgreSQL

What It Does

JIMMY is a Python-built backend platform and autonomous AI agent on AWS Bedrock (Claude Haiku 4.5) that monitors a live Kubernetes cluster, detects seven classes of infrastructure incidents, and executes a full 7-step remediation workflow without human intervention. The automated workflow is designed to reduce manual log triage from approximately 20 minutes to under 1 minute per incident, with post-incident summaries delivered via SNS.

Backend & API Architecture

  • 8 AWS Lambda functions (7 action groups plus an EventBridge-triggered detector running every 5 minutes) built in Python with boto3, exposing operational data and remediation actions via OpenAPI 3.0 specs covering logs, metrics, health checks, and pod scaling
  • In-memory caching implemented across action groups to reduce redundant API calls
  • 8 runbooks stored in S3 for deterministic runbook-first remediation logic
  • Streamlit dashboard in Python for real-time querying of system state and incident history
  • PostgreSQL for incident persistence, Node.js and React for the frontend layer, Anthropic API for natural language operational queries

Incident Types Handled

CrashLoopBackOff, OOMKilled, ImagePullBackOff, Readiness Probe Failure, High Restart Count, Pending Too Long, Service No Endpoints.

Remediation Workflow

ASSESS → DIAGNOSE → CLASSIFY → RUNBOOK → REMEDIATE → VERIFY → REPORT — every step executed autonomously by the agent against live cluster state.

Infrastructure & Observability

  • Autoscaling AWS EKS cluster (1–2 nodes) provisioned via Terraform and Helm
  • GitHub Actions CI across 7 Docker images with matrix builds (66% faster than sequential)
  • Closed-loop GitOps via ArgoCD and ArgoCD Image Updater
  • Full observability stack: Prometheus, Grafana, Fluent Bit, CloudWatch

YOLOv5 Real-Time Object Detection
Python · ML · Kubernetes · Springer ICCCE 2023

What It Does

A real-time object detection web application engineered in Python using YOLOv5 on the 80-class COCO dataset with Streamlit for live inference. The system achieved a 40% accuracy improvement over the baseline model. Research on this system was accepted and published in Springer ICCCE 2023 conference proceedings.

System Design

Built with a deliberate layered architecture separating the model inference layer, image preprocessing pipeline, and Streamlit presentation layer. Each layer is independently testable, independently replaceable, and independently deployable. This is software engineering discipline applied to an ML system, not just a model wrapped in a demo.

CI/CD Pipeline

  • Containerized with Docker; GitHub Actions CI/CD pipeline tagging each image with both latest and commit SHA for full traceability across all deployments
  • Deployed to a multi-node Kubernetes cluster on AWS EC2 via kubeadm with control plane and worker node isolation

Production Operations

  • Liveness and readiness probes configured for all pods
  • Live EBS volume expansion performed to resolve disk pressure with zero downtime
